## ⚡ 3步快速部署

### 第1步：解压文件

**Mac/Linux**：
```bash
tar -xzf storyboard-optimized-final.tar.gz
cd storyboard-optimized-final
```

**Windows**：
- 双击 `storyboard-optimized-final.zip` 解压

---

### 第2步：配置 Firebase（5分钟）⚠️ 必需

#### 2.1 创建 Firebase 项目

1. 访问 https://console.firebase.google.com/
2. 点击"添加项目"
3. 输入项目名称（如 `storyboard-review`）
4. 关闭 Google Analytics（可选）
5. 点击"创建项目"

#### 2.2 启用 Realtime Database

1. 在左侧菜单点击"构建" → "Realtime Database"
2. 点击"创建数据库"
3. 选择数据库位置（建议选 `us-central1` 或 `asia-southeast1`）
4. 选择"以测试模式启动"（临时使用）

#### 2.3 获取配置信息

1. 点击项目设置（左上角齿轮图标）
2. 滚动到"你的应用"区域
3. 点击"</>图标"（网页应用）
4. 注册应用，复制 `firebaseConfig` 对象

#### 2.4 填写配置文件

编辑 `firebase-config.js`，填入你的配置：

```javascript
const firebaseConfig = {
  apiKey: "AIzaSyDxxxxxxxxxxxxxxxxxxxxx",              // ← 替换这里
  authDomain: "your-project-id.firebaseapp.com",        // ← 替换这里
  databaseURL: "https://your-project-id.firebaseio.com", // ← 替换这里
  projectId: "your-project-id",                         // ← 替换这里
  storageBucket: "your-project-id.appspot.com",         // ← 替换这里
  messagingSenderId: "123456789012",                    // ← 替换这里
  appId: "1:123456789012:web:abc123def456"              // ← 替换这里
};
```

**⚠️ 重要**：不要使用 `YOUR_API_KEY` 这样的占位符，必须填写真实值！

## 🆘 常见问题

### Q1: 点击分享按钮没反应？
**A**: 检查 Firebase 配置是否正确填写。打开浏览器开发者工具（F12）查看错误信息。

### Q2: 分享链接打不开？
**A**: 确认：
- Firebase 已启用 Realtime Database
- 数据库规则设置为"测试模式"
- 链接完整复制（包含 `?room=xxx&mode=edit`）

### Q3: 实时同步不工作？
**A**: 检查：
- 网络连接是否正常
- Firebase 配置是否正确
- 浏览器控制台是否有报错

### Q4: 下载的 JSON 文件太大？
**A**:
- 压缩参考图（推荐 < 500 KB/张）
- 压缩 PDF 文件
- 删除不需要的参考图

### Q5: JSON 导入失败？
**A**:
- 确认 JSON 文件格式正确
- 检查文件是否损坏
- 尝试重新下载备份
