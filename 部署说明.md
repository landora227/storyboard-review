## 🚀 快速开始

### 第一步：配置 Firebase

1. **访问 Firebase Console**
   - 打开 [https://console.firebase.google.com/](https://console.firebase.google.com/)
   - 登录您的 Google 账号

2. **创建项目**
   - 点击"添加项目"
   - 输入项目名称（如：storyboard-review）
   - 根据需要选择是否启用 Google Analytics
   - 点击"创建项目"

3. **启用 Realtime Database**
   - 在项目控制台左侧菜单中，找到"构建" → "Realtime Database"
   - 点击"创建数据库"
   - 选择数据库位置（推荐选择离用户最近的区域）
   - 选择安全规则模式：
     - **测试模式**（推荐用于开发）：允许所有读写
     - **锁定模式**：需要手动配置规则
   - 点击"启用"

4. **配置安全规则**（可选但推荐）

   在 Realtime Database 的"规则"标签中，设置以下规则：

   ```json
   {
     "rules": {
       "rooms": {
         "$roomId": {
           ".read": true,
           ".write": true,
           ".indexOn": ["updatedAt"]
         }
       }
     }
   }
   ```

   这个规则允许：
   - 所有人都可以读取和写入房间数据
   - 如果需要更严格的权限控制，请参考 Firebase 文档自定义规则

5. **获取配置信息**
   - 点击项目设置（齿轮图标）→ "项目设置"
   - 滚动到"您的应用"部分
   - 点击"</>"图标（Web 应用）
   - 输入应用昵称（如：storyboard-app）
   - 复制配置代码

6. **更新 firebase-config.js**

   打开 `firebase-config.js` 文件，将配置信息替换为您的实际配置：

   ```javascript
   const firebaseConfig = {
     apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXX",
     authDomain: "your-project.firebaseapp.com",
     databaseURL: "https://your-project.firebaseio.com",
     projectId: "your-project",
     storageBucket: "your-project.appspot.com",
     messagingSenderId: "123456789012",
     appId: "1:123456789012:web:abcdef1234567890"
   };
   ```

### 第二步：部署文件

#### 方案 A：本地测试（使用 Live Server）

1. **安装 VS Code 扩展**
   - 安装 "Live Server" 扩展

2. **启动服务器**
   - 右键点击 `index-optimized.html`
   - 选择 "Open with Live Server"
   - 浏览器会自动打开应用

#### 方案 B：部署到 Web 服务器

1. **准备文件**

   将以下文件上传到您的 Web 服务器：
   ```
   index-optimized.html (重命名为 index.html)
   firebase-config.js
   collab-sync.js
   app-patch.js
   original-app.js
   original-styles.css
   original-archive-db.js
   ```

2. **上传到服务器**
   - 使用 FTP/SFTP 工具上传所有文件
   - 确保文件权限正确（一般设置为 644）

3. **访问应用**
   - 打开 `https://your-domain.com/index.html`

#### 方案 C：使用 GitHub Pages

1. **创建 GitHub 仓库**
   - 在 GitHub 上创建新仓库

2. **上传文件**
   - 将所有文件推送到仓库
   - 注意：将 `index-optimized.html` 重命名为 `index.html`

3. **启用 GitHub Pages**
   - 进入仓库设置（Settings）
   - 找到 "Pages" 选项
   - 选择部署分支（通常是 main）
   - 点击 "Save"

4. **访问应用**
   - GitHub 会提供一个 URL，如：`https://username.github.io/repo-name/`

#### 方案 D：使用 Netlify/Vercel

1. **安装 Netlify CLI**（可选）
   ```bash
   npm install -g netlify-cli
   ```

2. **部署**
   ```bash
   netlify deploy --dir=.
   ```

3. **或者使用拖拽部署**
   - 访问 [Netlify Drop](https://app.netlify.com/drop)
   - 将整个文件夹拖入页面
   - 获得部署链接

## ✨ 功能说明

### 1. 审核分享（编辑模式）

**使用场景**：团队协作，多人同时编辑和审核

**操作步骤**：
1. 导入 PDF 分镜
2. 点击"审核分享"按钮
3. 链接自动复制到剪贴板
4. 将链接分享给团队成员
5. 所有人的修改会实时同步

**特点**：
- 实时同步修改
- 显示在线用户数
- 支持多人同时编辑

### 2. 阅读分享（只读模式）

**使用场景**：向客户或相关方展示审核结果

**操作步骤**：
1. 完成审核工作
2. 点击"阅读分享"按钮
3. 链接自动复制到剪贴板
4. 分享链接给需要查看的人

**特点**：
- 只能查看，不能编辑
- 适合最终成果展示
- 保护数据不被误修改

### 3. 下载备份

**使用场景**：保存工作进度，离线存档

**操作步骤**：
1. 完成部分或全部审核工作
2. 点击"下载备份"按钮
3. 系统自动生成 JSON 文件：`PDF名称_MMDD.json`
4. 保存到本地

**文件命名规则**：
- 格式：`文件名_月日.json`
- 示例：`第一集分镜_1225.json`

**备份内容**：
- PDF 文件（Base64 编码）
- 所有参考图
- 所有文字反馈
- 时间戳

### 4. 导入备份

**使用场景**：恢复之前的工作进度

**操作步骤**：
1. 点击"导入备份"按钮
2. 选择之前下载的 JSON 文件
3. 系统自动恢复所有数据

**注意事项**：
- 导入会覆盖当前所有数据
- 请先备份当前工作再导入
- 确保 JSON 文件完整且未损坏

## 🛠️ 故障排除

### 问题 1：Firebase 初始化失败

**症状**：
- 控制台显示 "Firebase 初始化失败"
- 分享按钮点击后提示 "Firebase 未配置"

**解决方法**：
1. 检查 `firebase-config.js` 中的配置是否正确
2. 确认 Firebase 项目已启用 Realtime Database
3. 检查浏览器控制台的详细错误信息
4. 确认网络连接正常，能访问 Firebase 服务

### 问题 2：分享链接无法访问

**症状**：
- 链接可以复制，但打开后显示错误
- 提示 "找不到房间数据"

**解决方法**：
1. 确认 Firebase Realtime Database 安全规则允许读写
2. 检查链接格式是否正确：`?room=xxx&mode=edit`
3. 在 Firebase Console 中查看数据是否成功保存
4. 检查房间数据是否过期（默认不会过期）

### 问题 3：实时同步不工作

**症状**：
- 多人协作时，修改不会同步到其他人
- 只能看到自己的修改

**解决方法**：
1. 检查所有人是否使用同一个房间链接
2. 确认 Firebase Realtime Database 连接正常
3. 检查浏览器控制台是否有 WebSocket 连接错误
4. 刷新页面重新加入房间

### 问题 4：PDF 无法加载

**症状**：
- 导入 PDF 后页面空白
- 控制台显示 PDF.js 错误

**解决方法**：
1. 确认 PDF 文件没有损坏
2. 检查 PDF 文件大小（过大可能导致超时）
3. 尝试使用不同的 PDF 文件测试
4. 检查浏览器是否支持 PDF.js

### 问题 5：图片无法显示

**症状**：
- 上传图片后只显示占位符
- 导入备份后图片丢失

**解决方法**：
1. 检查图片格式（支持：JPG, PNG, GIF, WebP）
2. 检查图片大小（过大可能导致问题）
3. 确认图片的 base64 编码正确
4. 检查浏览器控制台的跨域错误

### 问题 6：下载备份失败

**症状**：
- 点击下载按钮没有反应
- 下载的 JSON 文件打不开

**解决方法**：
1. 检查浏览器是否阻止了下载
2. 查看浏览器下载设置
3. 尝试手动触发下载（右键保存）
4. 确认有足够的磁盘空间

```
rooms/
  └── room-xxx/
      ├── createdAt: 1234567890
      ├── updatedAt: 1234567890
      ├── pdf: "data:application/pdf;base64,..."
      ├── slots/
      │   ├── page-0-research-0/
      │   │   ├── text: "导演反馈文字"
      │   │   └── urls: ["data:image/png;base64,.."]
      │   ├── page-0-director-0/
      │   │   ├── text: "教研反馈文字"
      │   │   └── urls: ["data:image/png;base64,.."]
      │   └── ...
      └── users/
          ├── user-xxx/
          │   ├── id: "user-xxx"
          │   ├── joinedAt: 1234567890
          │   └── mode: "edit"
          └── user-yyy/
              ├── id: "user-yyy"
              ├── joinedAt: 1234567891
              └── mode: "read"
```

### 数据管理

**清理过期数据**：
1. 访问 Firebase Console
2. 进入 Realtime Database
3. 找到 `rooms/` 节点
4. 手动删除不需要的房间

**设置自动清理**（可选）：
```javascript
// 在 Firebase Functions 中设置
// functions/index.js
const functions = require('firebase-functions');
const admin = require('firebase-admin');
admin.initializeApp();

exports.cleanOldRooms = functions.pubsub
  .schedule('every 24 hours')
  .onRun(async (context) => {
    const db = admin.database();
    const roomsRef = db.ref('rooms');
    const snapshot = await roomsRef.once('value');
    const rooms = snapshot.val();

    const now = Date.now();
    const threeDays = 3 * 24 * 60 * 60 * 1000;

    for (const [roomId, room] of Object.entries(rooms)) {
      if (now - room.updatedAt > threeDays) {
        await roomsRef.child(roomId).remove();
        console.log(`Deleted room: ${roomId}`);
      }
    }
  });
```

## 🔐 安全建议

### 1. 配置 Firebase 安全规则

**推荐规则**（生产环境）：

```json
{
  "rules": {
    "rooms": {
      "$roomId": {
        ".read": "auth != null || data.exists()",
        ".write": "auth != null || !data.exists() || data.child('updatedAt').val() > now - 86400000",
        ".validate": "newData.hasChildren(['slots', 'pdf', 'updatedAt'])",
        "users": {
          "$userId": {
            ".write": "$userId === auth.uid || !data.exists()"
          }
        }
      }
    }
  }
}
```

这个规则：
- 允许未登录用户创建新房间
- 只允许修改 24 小时内的房间数据
- 用户只能修改自己的在线状态

### 2. 启用 Firebase Authentication（可选）

如果需要更严格的权限控制：

1. 在 Firebase Console 启用 Authentication
2. 选择登录方式（邮箱、Google、匿名等）
3. 修改 `firebase-config.js` 添加认证逻辑
4. 更新安全规则使用 `auth` 验证

### 3. 限制数据大小

在 Firebase Console 中设置：
- 单个写入操作限制：10MB
- 数据库总大小限制：根据套餐调整

### 4. 监控使用情况

定期检查：
- 数据库读写次数
- 存储空间使用
- 带宽消耗
- 异常访问模式

## 📈 性能优化

### 1. 图片优化

**建议**：
- 上传前压缩图片（推荐工具：TinyPNG）
- 限制单张图片大小（< 2MB）
- 使用 WebP 格式（更小的文件大小）

### 2. PDF 优化

**建议**：
- 压缩 PDF 文件（推荐工具：Adobe Acrobat）
- 限制 PDF 页数（< 50 页）
- 使用低分辨率版本用于在线协作

### 3. Firebase 优化

**建议**：
- 使用 `.indexOn` 优化查询
- 定期清理过期房间
- 考虑升级到 Blaze 套餐（按量付费）

### 4. 网络优化

**建议**：
- 启用 GZIP 压缩
- 使用 CDN 加速静态资源
- 启用浏览器缓存
