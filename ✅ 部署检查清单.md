## ⚙️ 第二阶段：配置 Firebase（5分钟）

### 创建 Firebase 项目
- [ ] 访问 https://console.firebase.google.com/
- [ ] 点击"添加项目"
- [ ] 输入项目名称：`storyboard-review`
- [ ] 关闭 Google Analytics（可选）
- [ ] 点击"创建项目"
- [ ] 等待项目创建完成

### 启用 Realtime Database
- [ ] 在左侧菜单找到"构建" → "Realtime Database"
- [ ] 点击"创建数据库"
- [ ] 选择数据库位置：
  - [ ] `us-central1`（美国）或
  - [ ] `asia-southeast1`（亚洲）
- [ ] 选择安全规则："以测试模式启动"
- [ ] 点击"启用"

### 获取配置信息
- [ ] 点击项目设置（左上角齿轮图标 ⚙️）
- [ ] 滚动到"你的应用"区域
- [ ] 点击"</>图标"（网页应用）
- [ ] 输入应用昵称：`storyboard-web`
- [ ] 不勾选"Firebase Hosting"
- [ ] 点击"注册应用"
- [ ] **复制** `firebaseConfig` 对象的内容

### 填写配置文件
- [ ] 用文本编辑器打开 `firebase-config.js`
- [ ] 找到这几行：
  ```javascript
  apiKey: "YOUR_API_KEY",
  authDomain: "YOUR_PROJECT.firebaseapp.com",
  databaseURL: "https://YOUR_PROJECT.firebaseio.com",
  projectId: "YOUR_PROJECT",
  storageBucket: "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID"
  ```
- [ ] **替换**为你从 Firebase 复制的真实值
- [ ] **保存文件**

**⚠️ 重要**：必须填写真实的 Firebase 配置值，不能是 `YOUR_API_KEY` 这样的占位符！

### 验证配置（可选但推荐）
- [ ] 检查 `apiKey` 是否以 `AIzaSy` 开头
- [ ] 检查 `projectId` 是否和你创建的项目名称一致
- [ ] 检查 `databaseURL` 是否包含正确的区域代码

## ⚠️ 常见问题排查

### 如果分享按钮没反应
- [ ] 检查浏览器控制台（F12）是否有错误
- [ ] 检查 `firebase-config.js` 配置是否正确
- [ ] 确认 Firebase Realtime Database 已启用
- [ ] 刷新页面重试

### 如果实时同步不工作
- [ ] 检查网络连接
- [ ] 检查 Firebase 配置
- [ ] 查看控制台报错信息
- [ ] 尝试重新生成链接

### 如果页面显示错误
- [ ] 检查是否所有文件都复制到项目目录
- [ ] 确认 `index.html` 文件名正确（不是 index-optimized.html）
- [ ] 检查 Firebase SDK 是否加载成功
- [ ] 清除浏览器缓存后刷新

### 如果 Git 推送失败
- [ ] 检查是否有权限
- [ ] 尝试先 `git pull` 再 `git push`
- [ ] 检查远程仓库地址是否正确
